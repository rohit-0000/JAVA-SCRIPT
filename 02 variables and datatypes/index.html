JavaScript Variable Rules

  1. Variable declaration:
    • Use let, const, or var.
    • let and const are block-scoped (modern), while var is function-scoped (older).
    • const is for constants that don't change their value.
    • var allows redeclaration of the same variable within the same scope, which can lead to unexpected behavior.
  2. Variable name:
    • Must begin with a letter, underscore (_), or dollar sign ($).
    • Cannot start with a number.
    • Can include letters, digits, underscores, and dollar signs.
    • Case-sensitive (e.g., myVar and myvar are different).
  3. Reserved words: Cannot use JavaScript keywords like class, return, function, etc., as variable names.

Example:


var x = 5;
var x = 10;  // No error, redeclaration is allowed
console.log(x);  // Output: 10

JavaScript Data Types

  1. Primitive Data Types:
    • Number: Represents both integers and floating-point numbers.
    • String: Represents sequences of characters.
    • Boolean: Represents true or false.
    • Undefined: A variable that has been declared but not assigned a value.
    • Null: Represents the intentional absence of any object value.
    • Symbol: Represents unique values often used as object keys.
    • BigInt: Used for large integers beyond the safe limit of Number.
  2. Non-Primitive Data Types (Reference Types):
    • Object: A collection of key-value pairs.
    • Array: An ordered list of values.
    • Function: A block of code designed to perform a task.
    • Date: Represents a specific moment in time.

Example:


let num = 42;  // Number
let name = "John";  // String
let isValid = true;  // Boolean
let nothing = null;  // Null
let bigIntValue = 9007199254740991n;  // BigInt
let obj = { key: "value" };  // Object
let arr = [1, 2, 3];  // Array
